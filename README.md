## 👨‍💻 About Me

I'm a backend-focused software engineer with experience designing, developing, deploying, and monitoring scalable applications.

My main stack is **C# and .NET**, with a strong interest in distributed systems, cloud-native architectures, microservices, infrastructure automation, and software reliability.

I enjoy working across the entire application lifecycle—from designing APIs and domain models to containerizing services, deploying them to Kubernetes, and implementing observability with metrics, logs, and distributed tracing.

* 🔧 Building backend applications with **C#, ASP.NET Core, EF Core, and Dapper**
* 🧩 Applying **DDD, Clean Architecture, CQRS, Event Sourcing, and SOLID**
* ☁️ Deploying cloud-native workloads with **Docker, Kubernetes, Helm, and Kustomize**
* 📡 Working with asynchronous communication using **RabbitMQ**
* 📊 Implementing observability with **OpenTelemetry, Prometheus, and Jaeger**
* 🗄️ Working with **PostgreSQL, SQL Server, and MySQL**
* 🚀 Exploring platform engineering, infrastructure as code, and site reliability practices
* 📚 Currently improving my knowledge of **Kubernetes, Terraform, distributed systems, and system design**

## 🚀 Featured Projects

### 📦 StockMode

A cloud-native inventory and sales platform built with a distributed architecture.

**Technologies:** .NET, ASP.NET Core, PostgreSQL, RabbitMQ, gRPC, Docker, Kubernetes, OpenTelemetry, Prometheus and Jaeger.

**Highlights:**

* Inventory, customer, sales, and stock movement management
* CQRS-based application architecture
* Asynchronous email processing with RabbitMQ
* QR code generation through a gRPC service
* Authentication and authorization with an identity provider
* Distributed tracing and application metrics
* Containerized services deployed to Kubernetes

### ⚡ Event-Sourcing Experiments

Experiments focused on modeling business processes using aggregates, commands, domain events, projections, and event streams.

**Technologies:** .NET, Marten, PostgreSQL, CQRS and Event Sourcing.

### ☸️ Kubernetes & Infrastructure Labs

* Deployments, Services, Ingresses and StatefulSets

* Helm and Kustomize
* AKS infrastructure
* Terraform modules
* CI/CD pipelines
